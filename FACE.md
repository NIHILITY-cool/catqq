# QQ 表情说明模板

这个文件是项目里维护 QQ 表情说明的单一来源。以后给 AI 的提示词、代码里的表情映射、文档里的表情列表，都应该先按这里改，再同步到对应位置。

当前推荐路线是：AI 只输出受控标记，例如 `[表情:比心]`；程序把这个标记转换成 AstrBot 的 `Face(id)` 组件。不要让 AI 直接输出 QQ 表情 id、CQ 码或任意收藏表情名。

## 维护原则

1. 只开放白名单里的表情。没有写进本文档的表情，不要写进人设，也不要让 AI 使用。
2. 给 AI 看的名字要稳定、短、好理解，例如 `比心`、`贴贴`、`耶`。不要用多个近义名字指向同一个表情。
3. 代码里使用数字 id，AI 提示词里使用名字。不要让 AI 记数字。
4. 一条回复最多 1 个表情，特殊撒娇或庆祝场景最多 2 个。不要连发一串。
5. 表情是语气补充，不是内容本身。重要提醒、道歉、拒绝、解释任务失败时少用或不用。
6. 跨账号带话时更克制。除非发起人明确要求可爱一点，否则不要把表情塞进别人要转达的正文。
7. 严肃场景不用表情：安全、健康、睡眠提醒、争吵、对方难过、任务执行错误、权限失败。
8. 新增或改名时同时更新：本文档、AI 人设里的可用表情说明、程序映射、测试。

## 推荐输出格式

AI 只能使用下面这种标记：

```text
[表情:比心]
```

标记可以放在句末，也可以单独成一段。不要写成：

```text
Face(319)
[CQ:face,id=319]
[表情:319]
比心表情
```

程序应该把 `[表情:比心]` 转换成 `Face(id=319)`，发出去后 QQ 里显示为真正的表情。

## 怎么得到表情 id

最稳的方式是从 AstrBot 日志读取 `Face(id)`。

1. 确认机器人正在运行。
2. 用白名单 QQ 给机器人发一个或几个目标表情。最好一次只发一个，方便对应名字。
3. 查看 AstrBot 最近日志：

```bash
docker compose logs --tail 200 astrbot
```

4. 找这种日志：

```text
Nihility/3262379680: [表情:318] [表情:319]
```

这里的 `318`、`319` 就是可用于 `Face(id=...)` 的 id。

5. 再看 NapCat 日志确认名字：

```bash
docker compose logs --tail 200 napcat
```

可能会看到：

```text
接收 <- 私聊 (3262379680) [表情 [崇拜]] [表情 [比心]]
```

把 AstrBot 的 id 和 NapCat 的名字按顺序对应起来，写进上面的表格。

注意：当前插件会拦截纯表情消息，不把它交给 LLM 回复；但 AstrBot 核心日志仍然能打印 `[表情:id]`，所以仍可用于采集 id。如果日志里看不到名字，就让发送人一次只发一个，并人工命名。

## 给 AI 的说明模板

把下面这一段按需加入 `persona.md`。不要整段机械照搬；要根据实际开放的表情表删减。

```text
## QQ 表情使用规则

你可以偶尔使用 QQ 表情，但只能使用下面这些受控标记：

- [表情:比心]：撒娇、感谢、喜欢时少量使用
- [表情:贴贴]：亲近、安慰时少量使用
- [表情:耶]：开心、完成小事、庆祝时少量使用
- [表情:打招呼]：刚开始聊天、轻松开场时使用

表情是语气补充，不是正文。大多数回复不用表情。一条回复最多 1 个表情，特别开心时最多 2 个。

不要输出表情 id，不要输出 CQ 码，不要写 Face(...)。如果没有合适表情，就只正常说话。

严肃提醒、拒绝别人、道歉、任务失败、对方难过、跨账号正式带话时，不要随便加表情。
```

## 程序映射模板

如果实现解析层，建议用名字到 id 的显式映射：

```python
QQ_FACE_ALIASES = {
    "崇拜": 318,
    "比心": 319,
    "打招呼": 341,
    "贴贴": 350,
    "干饭": 475,
    "对的对的": 478,
    "不对不对": 479,
    "右亲亲": 305,
    "耶": 355,
}
```

解析规则建议：

1. 只解析 `QQ_FACE_ALIASES` 里存在的名字。
2. 未知标记保留为普通文本或直接删除，但要写测试固定行为。
3. 单条消息最多转换 2 个表情，超过的标记保留文本或删除。
4. 输出消息链形如 `Plain + Face + Plain`，不要用纯文本 CQ 码做主路径。
5. 记忆里可以保留原始自然文本，也可以把表情标记保留成 `[表情:名字]`，但不要把数字 id 写进给 AI 的长期记忆。

## 新增表情条目模板

每新增一个表情，按这个格式填表：

```text
| 名字 | Face id | 建议用途 | 不建议用途 |
| --- | ---: | --- | --- |
| 贴贴 | 350 | 亲近、安慰、撒娇 | 对方明确不想被打扰时 |
```

同时补一条简短 AI 说明：

```text
- [表情:贴贴]：亲近、安慰时少量使用
```

再补程序映射：

```python
"贴贴": 350,
```

## 提交前检查

新增或修改 QQ 表情说明前，按这个清单检查：

- `FACE.md` 已更新
- `persona.md` 只引用本文档里已有的名字
- 程序映射只包含本文档里已有的名字和 id
- 测试覆盖了解析、未知表情、数量限制
- 已用真实 QQ 发过至少一个样本确认显示效果
