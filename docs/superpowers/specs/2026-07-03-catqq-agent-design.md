# CatQQ Agent Design Spec

**Date**: 2026-07-03
**Status**: Ready for implementation

## Overview

A QQ chatbot that talks like a tsundere cat ("小猫") using NapCatQQ + AstrBot + Docker Compose + DeepSeek V4.

### Architecture

```
QQ小号 (3757296370)
  ↓ QQ protocol
NapCatQQ (napcat container) — login, receive/send QQ messages
  ↓ OneBot v11 / Reverse WebSocket (ws://astrbot:6199/ws)
AstrBot (astrbot container) — AI orchestration, persona, plugins
  ↓ OpenAI-compatible API
DeepSeek V4 Flash (deepseek-v4-flash)
```

## Deployment

- **Environment**: Mac (local), Docker Desktop
- **Startup**: `NAPCAT_UID=$(id -u) NAPCAT_GID=$(id -g) docker compose up -d`
- **Management ports**: `6099` (NapCat WebUI), `6185` (AstrBot WebUI), both bound to `127.0.0.1`

## Project Files

Only five files need to be authored:

| File | Purpose |
|------|---------|
| `docker-compose.yml` | Two-container orchestration (NapCat + AstrBot) |
| `.env.example` | Environment variable template (whitelist, greeting times) |
| `.gitignore` | Exclude runtime data, secrets, QQ login state |
| `data/plugins/astrbot_plugin_cat_guard/metadata.yaml` | Plugin metadata |
| `data/plugins/astrbot_plugin_cat_guard/main.py` | Whitelist guard + sleep/wake + scheduled greetings |

Everything else (NapCat config, QQ login state, AstrBot data) is generated by the containers at runtime.

### .env.example

Whitelist QQ numbers are read from an environment variable to avoid hardcoding personal data:

```bash
# 白名单 QQ 号，逗号分隔
CATQQ_ALLOWED_USERS=3262379680,1906310787
# 早安时间（24小时制）
CATQQ_MORNING_HOUR=8
# 晚安时间（24小时制）
CATQQ_NIGHT_HOUR=23
```

On first use, copy `.env.example` to `.env` and edit the values. The `.env` file is git-ignored.

## Git and Distribution

The repo is designed to be clone-and-run. Someone else cloning it follows:

```bash
git clone <repo>
cd catqq-agent
cp .env.example .env
# Edit .env with your own QQ numbers
NAPCAT_UID=$(id -u) NAPCAT_GID=$(id -g) docker compose up -d
# Then: NapCat WebUI → scan QR, AstrBot WebUI → fill API Key
```

Personal data never enters git:
- QQ小号 → scan QR, session stored in git-ignored `ntqq/`
- API Key → filled in AstrBot WebUI, stored in git-ignored `data/config/`
- Whitelist QQ numbers → in `.env` (git-ignored), template in `.env.example` (committed)

## Docker Compose

Two services sharing `./data:/AstrBot/data` and an internal bridge network:

- **napcat**: `mlikiowa/napcat-docker:latest`, mode=astrbot, port `6099` on loopback
- **astrbot**: `soulter/astrbot:latest`, port `6185` on loopback

Both ports bound to `127.0.0.1` — no public exposure. On Mac, pass `NAPCAT_UID` and `NAPCAT_GID` from `id -u`/`id -g`.

## AI Model

- **Provider**: DeepSeek official API
- **Model**: `deepseek-v4-flash` (284B params, 1M context)
- **Config method**: AstrBot WebUI → Providers → OpenAI Compatible
  - Base URL: `https://api.deepseek.com`
  - API Key: configured via WebUI (not in files)
- **Note**: `deepseek-chat` alias will stop working 2026-07-24; use `deepseek-v4-flash` directly

## Cat Persona

Configured in AstrBot WebUI → Persona. The full persona prompt defines:
- Core identity: a tsundere cat named "玖玖" (Jiu Jiu) who treats the user as "自己的人" (her person). The name doesn't need to be mentioned often, but the cat knows its name.
- Calls the user "人" frequently
- Short conversational replies (1-3 sentences normally)
- Eleven chat-state rules (just arrived, long silence, busy, other cats, praise, sad, study/code, teased, confessed, told to stop)
- Punctuation rules: periods signal mood (jealous/tsundere/sulking); "呀/嘛/喵/～" for happy/coaxed
- "Chat screenshot vibe" — messages should feel like a viral WeChat/QQ screenshot
- If asked whether AI: "小猫是一个会自动聊天的小猫账号。但小猫会认真陪你说话。"

## Plugin: cat_guard

Single plugin (`astrbot_plugin_cat_guard`) handling three responsibilities:

### 1. Whitelist Guard
- Allowed users: read from `CATQQ_ALLOWED_USERS` env var (comma-separated QQ numbers)
- Default if unset: `3262379680,1906310787`
- Group messages: always blocked
- Non-whitelist private messages: `stop_event()`, no LLM call
- Whitelist private messages: pass through to AstrBot's default LLM flow

### 2. Sleep/Wake
- "小猫睡觉" → reply "咪睡了，人晚安", enter sleeping state
- "小猫醒醒" → reply in cat style, exit sleeping state
- While sleeping: all messages from whitelist users are blocked (no LLM)

### 3. Scheduled Greetings
- Good morning: 08:00 daily
- Good night: 23:00 daily
- Sent to all whitelist users via AstrBot message API
- Content in cat persona style
- Implemented as an async loop checking time every 60 seconds

## Integration Flow

```
NapCat QQ message received
  → OneBot v11 → ws://astrbot:6199/ws
  → AstrBot receives event
  → cat_guard plugin fires:
      1. Group? → block
      2. Non-whitelist? → block
      3. "小猫睡觉"? → reply + sleep
      4. "小猫醒醒"? → reply + wake
      5. Sleeping? → block
      6. Else → pass to LLM with cat persona
  → AstrBot sends reply back via OneBot v11
  → NapCat delivers QQ message
```

## Connection Setup

1. NapCat WebUI (6099): scan QR with QQ小号 to login
2. AstrBot WebUI (6185): create OneBot v11 bot, Reverse WS on `0.0.0.0:6199`, path `/ws`
3. NapCat WebUI: add WebSocket Client → `ws://astrbot:6199/ws`
4. Verify: AstrBot console shows "aiocqhttp(OneBot v11) adapter connected"
